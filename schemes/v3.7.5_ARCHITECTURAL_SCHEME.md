# v3.7.5 Architectural Loop Scheme [ABSOLUTE_VERIFICATION]

This scheme visually maps the project's dual-mode autonomous engine.

```mermaid
graph TD
    Start((Session Start)) --> WAV{WAV Gate}
    WAV -- Fail --> Stop((Reset))
    WAV -- Pass --> SelectMode[Mode Selection]

    SelectMode -->|/run| BroadLoop[Broad Loop: Multi-Task Sprint]
    SelectMode -->|/deep_run| DeepLoop[Deep Loop: Surgical Hardening]

    subgraph "Broad Loop [/run]"
        BroadLoop --> PlanA[Surgical Plan]
        PlanA --> ExecA[Fast Execution]
        ExecA --> TestA[/test]
        TestA --> Next{Next TODO?}
        Next -->|Yes| PlanA
        Next -->|Limit Reached| CycleEnd
    end

    subgraph "Deep Loop [/deep_run]"
        DeepLoop --> Route[Test Route Planning]
        Route --> Research[Trio-Path Research]
        Research --> ExecB[Execute Best Fix]
        ExecB --> Rebuild[Mandatory APK Rebuild]
        Rebuild --> DeepTest[/deep_test]
        DeepTest -- Fail --> IterCheck{Cap < 4?}
        IterCheck -- Yes --> Research
        IterCheck -- No --> Escalate((Manual Escalate))
        DeepTest -- Pass --> CycleEnd
    end

    CycleEnd[Cycle Closure] --> Logit[/logit Sync]
    Logit --> Git[Git Commit]
    Git --> Done((Session Complete))

    style DeepLoop fill:#f96,stroke:#333,stroke-width:4px
    style DeepTest fill:#f96,stroke:#333,stroke-width:2px
    style BroadLoop fill:#96f,stroke:#333,stroke-width:2px
```

## 🛡️ v3.7.5 Hardening Gates
1. **[GATE_1] Test Route**: Define the "Wanted Result" exactly BEFORE coding.
2. **[GATE_2] Trio-Path**: Research 3 distinct fix options before the first edit.
3. **[GATE_3] Rebuild Mandate**: Any core fix requires a full `flutter build apk` session.
4. **[GATE_4] Collision Check**: Visual proof must show the fix color persists WITHOUT active selection handles.
5. **[GATE_5] Iteration Cap**: Max 4 loops to prevent thashing.
